1. An adversary can gain unauthorized access to Azure Key Vault instances due to weak network security configuration.  [State: Mitigation Implemented]  [Priority: High] 

Category:Elevation of Privileges
Description:An adversary can gain unauthorized access to Azure Key Vault instances due to weak network security configuration.
Justification:<no mitigation provided>
Short Description:A user subject gains increased capability or privilege by taking advantage of an implementation bug
Possible Mitigation(s):Restrict access to Azure Key Vault instances by configuring firewall rules to permit connections from selected networks (e.g. a virtual network or a custom set of IP addresses).For Key Vault client applications behind a firewall trying to access a Key Vault instance, see best practices mentioned here: <a href="https://aka.ms/tmt-th179 ">https://aka.ms/tmt-th179 </a>
SDL Phase:Implementation

2. An adversary can deny actions performed on Azure Key Vault due to a lack of auditing.   [State: Mitigation Implemented]  [Priority: Medium] 

Category:Repudiation
Description:An adversary can deny actions performed on Azure Key Vault due to a lack of auditing.
Justification:<no mitigation provided>
Short Description:Repudiation threats involve an adversary denying that something happened
Possible Mitigation(s):Enable audit logging on Azure Key Vault instances to monitor how and when the instances are access, and by whom. Use standard Azure access controls to restrict access to the logs. Refer : <a href="https://aka.ms/tmt-th180 ">https://aka.ms/tmt-th180 </a>
SDL Phase:Implementation

3. An adversary may gain unauthorized access to manage Azure Key Vault due to weak authorization rules.  [State: Mitigation Implemented]  [Priority: High] 

Category:Elevation of Privileges
Description:An adversary may gain unauthorized access to manage Azure Key Vault due to weak authorization rules.
Justification:<no mitigation provided>
Short Description:A user subject gains increased capability or privilege by taking advantage of an implementation bug
Possible Mitigation(s):Access to the Azure Key Vault management plane should be restricted by choosing appropriate Role-Based Access Control (RBAC) roles and privileges in accordance with the principle of least privilege. Over permissive or weak authorization rules may potentially permit data plane access (e.g. a user with Contribute (RBAC) permissions to Key Vault management plane may grant themselves access to the data plane by setting the Azure Key Vault access policy). Refer : <a href="https://aka.ms/tmt-th181 ">https://aka.ms/tmt-th181 </a>
SDL Phase:Implementation

5. An adversary can abuse poorly managed authentication/access policies. An adversary may gain unauthorized access to Azure Key Vault due to compromise of secret/certificate used to authenticate to Azure Key Vault .   [State: Mitigation Implemented]  [Priority: High] 

Category:Elevation of Privileges

Justification:<no mitigation provided>
Short Description:A user subject gains increased capability or privilege by taking advantage of an implementation bug
Possible Mitigation(s):Use managed identities for Azure resources and details can be found here at <a href="https://aka.ms/tmt-th183 ">https://aka.ms/tmt-th183 </a>. 11. An adversary can read sensitive data by sniffing or intercepting traffic to Azure App Service Web App  [State: Mitigation Implemented]  [Priority: High] 

Category:Tampering
Description:An adversary can read sensitive data by sniffing or intercepting traffic to Azure App Service Web App
Justification:<no mitigation provided>
Short Description:Tampering is the act of altering the bits. Tampering with a process involves changing bits in the running process. Similarly, Tampering with a data flow involves changing bits on the wire or between two running processes
Possible Mitigation(s):Configure SSL certificate for custom domain in Azure App Service. Force all HTTP traffic to the app service to be over HTTPS by enabling the HTTPS only option on the instance. Refer: <a href="https://aka.ms/tmt-th175c">https://aka.ms/tmt-th175c</a>
